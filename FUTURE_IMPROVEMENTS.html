Future Improvements

A roadmap of features and enhancements to build next.

Last updated April 2026
22 ideas across 6 categories
💬
Messaging
Improvements to the core chat experience
↩️
Easy

Reply to Messages

Tap any message to quote-reply to it. The reply shows a preview of the original message above the new one, making conversations easier to follow.

💡 Store parent message ID in the messages table
😄
Easy

Emoji Reactions

Let users react to messages with emoji (👍 ❤️ 😂 etc.). Show reaction counts beneath each message. Remove your own reaction by clicking it again.

💡 New reactions table linked to message IDs
✏️
Easy

Typing Indicators

Show a "Someone is typing…" indicator at the bottom of the chat when another user is composing a message. Auto-hide after a short idle timeout.

💡 Socket.io event, no DB change needed
🎨
Easy

Markdown Support

Let users format messages with basic Markdown: **bold**, _italic_, `code`, and fenced code blocks. Sanitize output to prevent XSS.

💡 marked.js + DOMPurify on the client
⏱️
Medium

Custom Self-Destruct Timers

Allow each room to have its own configurable message expiry — from 1 minute to 24 hours. Admins set it per room, users see the countdown on each message.

💡 Add ttl_seconds column to rooms table
🔍
Medium

Message Search

A search bar in each room that filters visible messages in real time by keyword. For admins, search across all rooms from the dashboard.

💡 SQLite FTS5 (full-text search) extension
👤
Users & Identity
Better user experience and identity options
🎭
Easy

Generated Avatars

Auto-generate a unique color avatar for each username using a hash — so the same username always gets the same color, making users visually recognizable without accounts.

💡 Hash username → HSL color, no storage needed
📩
Medium

Private Direct Messages

Let two online users open a private 1-on-1 chat. No message history kept — everything is ephemeral and disappears when either user disconnects.

💡 Private Socket.io rooms per user pair
🔔
Medium

Browser Notifications

Ask permission once, then send a Web Push notification when a new message arrives in a room the user has open in a background tab.

💡 Notification API — no server push needed for tab-level
🏠
Rooms
More flexibility in how rooms work
📂
Easy

Room Categories

Group rooms into categories (e.g. General, Gaming, Tech) shown as collapsible sections in the sidebar, making navigation easier as the number of rooms grows.

💡 Add category column to rooms table
📌
Easy

Pinned Messages

Admins can pin up to 3 messages per room. Pinned messages appear in a dismissible banner at the top of the chat, visible to all members on join.

💡 Add pinned flag to messages table
🔒
Medium

User Capacity Limits

Let admins set a maximum member count per room. When the room is full, new users see a "Room is full" message and can join a waitlist.

💡 Add max_users column to rooms table
⏰
Medium

Temporary Rooms

Create rooms that automatically delete themselves after a set time (e.g. 1 hour, 1 day). Useful for events, sessions, or one-off discussions.

💡 Add expires_at column, background cleanup job
🖼️
Media & Files
Sharing beyond plain text
🖼️
Medium

Image Sharing

Paste or upload images directly into chat. Show an inline thumbnail that users can click to expand. Images self-destruct with the message they're attached to.

💡 Store to /uploads, serve via Express static, or use Cloudinary free tier
🎤
Hard

Voice Messages

Hold a button to record a short audio clip (max 60 seconds) and send it as a message. The recipient sees a waveform and a play button inline in the chat.

💡 MediaRecorder API, store as .webm blob
🔗
Easy

Link Previews

When a message contains a URL, automatically fetch its Open Graph metadata (title, description, image) server-side and show a small preview card below the message.

💡 og-scraper on the server, cached in SQLite
🛡️
Security & Moderation
Keeping the platform safe
🚫
Easy

Word Filter

A configurable blocklist of words managed from the admin dashboard. Matching messages are either blocked or have the word replaced with asterisks before sending.

💡 Store blocklist in config table, check on server emit
⚡
Easy

Rate Limiting

Limit users to a maximum of N messages per minute per room to prevent spam and flooding. Show a countdown timer when the limit is hit.

💡 Per-socket message counter with sliding window
🕐
Medium

Timed Bans

Upgrade "kick" to support timed bans by IP address — 10 minutes, 1 hour, or permanent. The admin sets the duration when banning and the ban auto-expires.

💡 New bans table with expires_at, checked on socket connect
🔐
Hard

End-to-End Encryption

Encrypt messages in the browser before sending so the server never sees plaintext. Only users in the room with the shared key can decrypt and read them.

💡 Web Crypto API — ECDH key exchange per room
⚙️
Admin & Infrastructure
Better tooling, analytics, and deployment
📊
Easy

Live Analytics Charts

Add a charts panel to the admin dashboard showing messages per hour, peak concurrency, most active rooms, and new user joins over time.

💡 Recharts (already in the project) + aggregation queries
📤
Easy

Export Chat History

Admin can download the full message history for any room as a JSON or plain-text file for archiving or moderation review before deleting the room.

💡 Express endpoint streaming SQLite results as JSON
☁️
Medium

Persistent Cloud Database

Migrate from local SQLite files to Turso (hosted libSQL) so data survives server restarts, scaling events, and re-deploys on free hosting tiers.

💡 Turso free tier — 500 DBs, 9 GB, libSQL-compatible API
🎙️
Hard

Voice Channels (WebRTC)

Add optional voice-only channels alongside text rooms. Users join a voice channel to talk in real time using browser WebRTC with no external service required.

💡 simple-peer + Socket.io for WebRTC signaling
